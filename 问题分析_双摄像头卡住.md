# 双摄像头程序卡住问题分析

## 问题现象
运行 `run_backend_service.py` 时，程序卡住无响应

## 可能原因
程序尝试初始化**两个摄像头**（设备ID 0和1），如果只有1个摄像头，摄像头1初始化会失败或卡住

## 验证方法
检查可用的摄像头数量：

```bash
.venv\Scripts\python.exe -c "import cv2; [print(f'Camera {i} OK') for i in range(5) if cv2.VideoCapture(i).read()[0]]"
```

## 解决方案

### 方案1：使用单摄像头模式
使用 `run_with_rvm.py` 进行测试：
```bash
.venv\Scripts\python.exe run_with_rvm.py
```

### 方案2：修改配置让第二个摄像头复用第一个
如果只有一个摄像头，可以修改代码让两个摄像头都使用设备0

### 方案3：添加Unity通信到单摄像头模式
创建一个基于单摄像头的Unity通信测试版本

## 建议
先验证摄像头数量，然后根据实际情况选择方案
